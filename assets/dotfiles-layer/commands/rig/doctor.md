对**当前项目**跑 rig 自检,有问题就帮我修。

## 1. 自检
执行 `rig doctor`（或 `~/.claude/skills/rig/scripts/verify.sh "$PWD"`），原样汇报六段（注入 / 红线拦 / 红线放 / settings 注册 / 失败降级 / 新 hook 行为）。
全绿（`关键项全部通过`）→ 报"健康"，结束。

## 2. 有 ✗ → 诊断 + 开药（动手前先问我）
把每条 ✗ 映射到根因与修复动作（**先只读地**确认根因），列给我、**等我确认再改**：

| ✗ 症状 | 根因 | 修复动作 |
|---|---|---|
| 找不到 hook 脚本 / settings 未注册 | bootstrap 没跑全 | 重跑 `~/.claude/skills/rig/scripts/bootstrap.sh`（幂等） |
| 红线没拦 / 注入空 / 大面积 ✗ | **jq 没装**（hook 全静默空转，阻断级） | 按机器包管理器装 jq（`brew`/`apt install jq`）——**联网+安装，单独征我同意**；装完重跑 bootstrap |
| 装了却"没反应" | hook 改了没开新会话 | 提醒我**开新会话**（这条你改不了，只能我来） |
| 注入"无输出" / 提示无 lint-one | 本项目还没接入 | 跑 `/rig:init` 接入当前项目 |
| 失败降级 ✗（该放行却 exit 2 拦） | 项目 `scripts/lint-one.sh` 占位 / 检查器没配却硬跑 | 把对应语言分支改成未配时 `exit 0`（或填真实命令） |

## 铁律
- **先问后改**：任何写操作（重跑 bootstrap、装 jq、改 lint-one、跑 `/rig:init`）都先把动作讲清、等我确认；能只读诊断的绝不擅自改。
- 装 jq 等联网/安装，单独征得我同意。
- "开新会话"这类你改不了的，明确告诉我去做，**别假装修好了**。
- 修完**重跑一次自检**确认变绿，把前后对比给我。
